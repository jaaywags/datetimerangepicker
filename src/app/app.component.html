Your selected start date is: {{startDate}}
Your selected end date is: {{endDate}}
Implementing this package is simple. Just place the following in your html:
<lib-date-time-range-picker (onDatePicked)="setDateRange($event)" [startDate]="startDate" [endDate]="endDate"></lib-date-time-range-picker>

There are three attribute:
  1. (onDatePicked)="someFunction($event)
    • This function gets called when both dates are selected, and when one of the two dates change
    • The object passed into this function as two properties, startDate and endDate
    • This function needs to be in your component class and will look something like:
      public someFunction(dates: any): void {{ '{' }}
          this.startDate = dates.startDate;
          this.endDate = dates.endDate;
      {{ '}' }}
  2. [startDate]="componentVariableWithStartDate"
    • This can be an empty string initially, but when the date is emitted into the above function, it should get stored into a variable that goes into here. That way if you hide this component for some reason (like a loading icon), the dates will still be there when it displays again.
  3. [endDate]="componentVariableWithEndDate"
    • Same as above

Notes:
  • The date returned is in UTC
  • If you provide a start date or end date, they need to be in UTC
  • The date the user selects is assumed to be in their local time and will be converted to UTC
  • If you have the momentjs package, you can format the returned date easily for SQL. For me I format the date for sql server using the following:
    • moment(startDate).format('YYYY-MM-DD HH:mm:00.0000000')
  • To nicely format dates for the user, use the following:
    • moment(startDate).format('MMM Do, YYYY hh:mm:ss a')
